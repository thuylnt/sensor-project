 Quyết định kỹ thuật (chi tiết)

 1. Firmware ESP32 (C++ / Arduino-ESP32 hoặc ESP-IDF)

 - Sampling: 100 Hz, MPU6050 DLPF_CFG=3 (cutoff ~44 Hz) để chống aliasing.
 - Cửa sổ: 2s (200 mẫu) × 6 trục, overlap 50% → 1 inference/giây.
 - Tiền xử lý: low-pass Butterworth bậc 2 cutoff 10 Hz cho HAR; high-pass IIR cutoff 0.3 Hz để tách gravity; z-score per-window per-axis.
 - Phân chia task (FreeRTOS): sampling ISR + ring buffer; inference task ở core 1; WiFi/MQTT ở core 0; queue truyền window giữa các task.
 - Calibration: script bias + scale chạy 1 lần khi setup (6 mặt cho accel, 30s đứng yên cho gyro), lưu NVS flash, load khi boot.

 2. Model HAR

 - Kiến trúc 1D-CNN input (200, 6):
   - Conv1D(16, k=9, s=2) + ReLU + BN + MaxPool(2)
   - Conv1D(32, k=5, s=1) + ReLU + BN + MaxPool(2)
   - Conv1D(32, k=3) + GlobalAveragePool
   - Dense(32) + Dropout(0.3) + Dense(num_classes, softmax)
 - Số param: ~8–12K; sau int8 quantize ≈ 15–25 KB flash, ~30 KB tensor arena. Inference < 50 ms.
 - Classes khuyến nghị (4 class an toàn): stand, walk, run, stairs. Bỏ fall trừ khi có >200 mẫu té (rất khó thu).
 - Framework: Edge Impulse (Studio UI, auto-quantize, export thư viện Arduino sẵn — phù hợp đồ án sinh viên). Lựa chọn 2: TensorFlow + TFLite Micro nếu muốn tự control hoàn
 toàn.

 3. PDR-lite (sau khi HAR phát hiện walk/run)

 - Step detection: peak detection trên |accel| đã low-pass, kết hợp ngưỡng thời gian tối thiểu giữa 2 peak (~250 ms).
 - Stride length: Weinberg model k * (a_max - a_min)^(1/4), hệ số k calibrate bằng đường thẳng 20 m.
 - Heading: tích phân gyro Z, bias EMA động — khi HAR = stand thì update bias.
 - ZUPT: giữa 2 peak step coi như đứng → reset vận tốc tích lũy.
 - Reset trajectory: nút bấm reset trên dashboard, hoặc auto-reset sau 3 phút (giảm drift hiển thị).

 4. Server stack (docker-compose trên laptop sinh viên)

 - Mosquitto broker (port 1883, không TLS).
 - InfluxDB 2.x — time-series native, retention 30 ngày.
 - Telegraf với plugin mqtt_consumer → write thẳng InfluxDB (đơn giản hơn viết script).
 - Grafana — chart accel, step count theo giờ, activity timeline.
 - Node-RED — quỹ đạo 2D vẽ trên canvas SVG (hoặc node worldmap nếu muốn map thật), nút reset, hiển thị activity hiện tại.

 5. Topic MQTT

 usth/pdr/<device_id>/activity   {ts, class, confidence}      # 1 Hz
 usth/pdr/<device_id>/step       {ts, count, cadence}         # mỗi bước
 usth/pdr/<device_id>/pose       {ts, x, y, heading_deg}      # 1–2 Hz
 usth/pdr/<device_id>/status     {battery, rssi, uptime}      # retained, 0.1 Hz
 usth/pdr/<device_id>/raw        {ts, ax,ay,az,gx,gy,gz}      # CHỈ bật khi collect data
 Không stream raw 100 Hz qua MQTT trong chế độ vận hành — quá tải broker và mất ý nghĩa edge AI.

 6. Ground truth & data collection

 - Người: 3–5 người (đa chiều cao/dáng đi), 2–3 phiên khác ngày.
 - Mỗi class: 5–10 phút/người/phiên → mục tiêu ~30–60 phút/class sau cửa sổ hóa ≈ 1800–3600 windows/class.
 - 2 vị trí gắn sensor: thắt lưng (chính) + túi quần (test generalization).
 - Tool label: script Python đọc serial CSV, phím nóng 1=stand 2=walk 3=run 4=stairs space=pause. Ghi label đồng bộ theo timestamp ESP32.
 - Đánh giá PDR: đi đường vuông 10 m × 5 m, lặp 3–5 vòng, đo closure error (khoảng cách điểm cuối – điểm đầu) và % so với tổng quãng đường. Mục tiêu 5–15% drift / 100 m là tốt
  cho 6-axis IMU không mag.

     ---
     Quyết định kỹ thuật (chi tiết)

     1. Firmware ESP32 (C++ / Arduino-ESP32 hoặc ESP-IDF)

     - Sampling: 100 Hz, MPU6050 DLPF_CFG=3 (cutoff ~44 Hz) để chống aliasing.
     - Cửa sổ: 2s (200 mẫu) × 6 trục, overlap 50% → 1 inference/giây.
     - Tiền xử lý: low-pass Butterworth bậc 2 cutoff 10 Hz cho HAR; high-pass IIR cutoff 0.3 Hz để tách gravity; z-score per-window per-axis.
     - Phân chia task (FreeRTOS): sampling ISR + ring buffer; inference task ở core 1; WiFi/MQTT ở core 0; queue truyền window giữa các task.
     - Calibration: script bias + scale chạy 1 lần khi setup (6 mặt cho accel, 30s đứng yên cho gyro), lưu NVS flash, load khi boot.

     2. Model HAR

     - Kiến trúc 1D-CNN input (200, 6):
       - Conv1D(16, k=9, s=2) + ReLU + BN + MaxPool(2)
       - Conv1D(32, k=5, s=1) + ReLU + BN + MaxPool(2)
       - Conv1D(32, k=3) + GlobalAveragePool
       - Dense(32) + Dropout(0.3) + Dense(num_classes, softmax)
     - Số param: ~8–12K; sau int8 quantize ≈ 15–25 KB flash, ~30 KB tensor arena. Inference < 50 ms.
     - Classes khuyến nghị (4 class an toàn): stand, walk, run, stairs. Bỏ fall trừ khi có >200 mẫu té (rất khó thu).
     - Framework: Edge Impulse (Studio UI, auto-quantize, export thư viện Arduino sẵn — phù hợp đồ án sinh viên). Lựa chọn 2: TensorFlow + TFLite Micro nếu muốn tự control hoàn
      toàn.

     3. PDR-lite (sau khi HAR phát hiện walk/run)

     - Step detection: peak detection trên |accel| đã low-pass, kết hợp ngưỡng thời gian tối thiểu giữa 2 peak (~250 ms).
     - Stride length: Weinberg model k * (a_max - a_min)^(1/4), hệ số k calibrate bằng đường thẳng 20 m.
     - Heading: tích phân gyro Z, bias EMA động — khi HAR = stand thì update bias.
     - ZUPT: giữa 2 peak step coi như đứng → reset vận tốc tích lũy.
     - Reset trajectory: nút bấm reset trên dashboard, hoặc auto-reset sau 3 phút (giảm drift hiển thị).

     4. Server stack (docker-compose trên laptop sinh viên)

     - Mosquitto broker (port 1883, không TLS).
     - InfluxDB 2.x — time-series native, retention 30 ngày.
     - Telegraf với plugin mqtt_consumer → write thẳng InfluxDB (đơn giản hơn viết script).
     - Grafana — chart accel, step count theo giờ, activity timeline.
     - Node-RED — quỹ đạo 2D vẽ trên canvas SVG (hoặc node worldmap nếu muốn map thật), nút reset, hiển thị activity hiện tại.

     5. Topic MQTT

     usth/pdr/<device_id>/activity   {ts, class, confidence}      # 1 Hz
     usth/pdr/<device_id>/step       {ts, count, cadence}         # mỗi bước
     usth/pdr/<device_id>/pose       {ts, x, y, heading_deg}      # 1–2 Hz
     usth/pdr/<device_id>/status     {battery, rssi, uptime}      # retained, 0.1 Hz
     usth/pdr/<device_id>/raw        {ts, ax,ay,az,gx,gy,gz}      # CHỈ bật khi collect data
     Không stream raw 100 Hz qua MQTT trong chế độ vận hành — quá tải broker và mất ý nghĩa edge AI.

     6. Ground truth & data collection

     - Người: 3–5 người (đa chiều cao/dáng đi), 2–3 phiên khác ngày.
     - Mỗi class: 5–10 phút/người/phiên → mục tiêu ~30–60 phút/class sau cửa sổ hóa ≈ 1800–3600 windows/class.
     - 2 vị trí gắn sensor: thắt lưng (chính) + túi quần (test generalization).
     - Tool label: script Python đọc serial CSV, phím nóng 1=stand 2=walk 3=run 4=stairs space=pause. Ghi label đồng bộ theo timestamp ESP32.
     - Đánh giá PDR: đi đường vuông 10 m × 5 m, lặp 3–5 vòng, đo closure error (khoảng cách điểm cuối – điểm đầu) và % so với tổng quãng đường. Mục tiêu 5–15% drift / 100 m là
     tốt cho 6-axis IMU không mag.
